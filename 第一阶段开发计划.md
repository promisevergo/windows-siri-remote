# 第一阶段开发计划

## 概要

- 使用 .NET 10、C# 和 `net10.0-windows10.0.19041.0` 建立 `RemoteMapper.Cli`、`RemoteMapper.Core`、`RemoteMapper.Tests` 三项目方案。
- 先完成实体 A2854 可行性探针；只有按键、单双指坐标和圆环连续数据全部通过，才继续完整映射开发。
- 保持 Apache-2.0：仅借鉴参考项目中的协议事实，C# 代码与测试独立实现，不复制 GPLv3 源码。
- 本轮在当前 Windows 10 22H2 上开发和验收；Windows 11 明确标记为尚未实测。

## 实现内容

### 1. 工具链与可行性门槛

- 安装官方 .NET 10 SDK；更新 `.gitignore`，忽略构建产物、本地工具和 `参考项目/`，不改动参考项目。
- 增加 `RemoteMapper.exe --probe`：
  - 枚举已配对 BLE 设备，识别 HID 服务 `0x1812`。
  - 枚举全部 `0x2A4D` Report 特征，读取 `0x2908` Report Reference。
  - 先订阅 Input Report，再向可写 Output/Feature Report 写入 `0xAF`。
  - 验证 `0xFB` 两字节按键掩码、`0xFC` 11/18 字节单双指触摸报告、连续坐标与旋转轨迹。
  - 交互式提示用户依次测试全部实体按键、单指中心滑动、双指圆环轻碰和正反向旋转。
- 若 WinRT GATT 无法访问，执行一次受限的 Win32 HID 原始读取验证；两条用户态路径都失败时停止完整开发，输出门槛失败记录。
- 根据本机采样冻结触摸范围、圆环边界、抖动死区和滚轮角度步长；双指时间阈值默认保持需求中的 100/250/100 ms 和 5% 距离限制。

### 2. 协议、状态机与 Windows 输出

- Core 定义统一的按键、触摸、连接事件，以及键盘、鼠标和滚轮输出动作。
- 纯状态机通过 `Process(event, timestamp)`、`Tick(timestamp)` 和 `Reset()` 产生输出，不直接依赖蓝牙或 `SendInput`。
- 独立实现按键掩码、12 位有符号坐标、单双指触点和触摸结束帧解析。
- 完成需求映射：
  - 中心键短按左击、连续短按由 Windows 识别为双击、1 秒长按发送一次 Enter。
  - TV 键短按 Alt+Tab；1 秒长按进入鼠标左键拖拽，松开或断线时释放。
  - 方向键按下/松开及长按重复；重复参数读取 Windows 当前设置。
  - Delete、静音、音量增减和音量长按重复。
  - 中心起点锁定鼠标移动；圆环起点识别滚轮；累计旋转角度后输出滚轮步进。
  - 双指圆环轻碰严格执行时间、距离、区域和物理按键排他规则。
- 使用 Win32 `SendInput` 产生相对鼠标、滚轮、键盘和媒体键操作，并集中跟踪、释放所有已按下状态。

### 3. 控制台、配置与生命周期

- 对外命令：
  - `RemoteMapper.exe`：读取配置并开始映射。
  - `--list-devices`：列出候选设备后退出。
  - `--verbose`：输出解码后的调试事件。
  - `--probe`：运行第一阶段硬件验证。
- `%LOCALAPPDATA%\RemoteMapper\config.json` 保存设备 ID、名称、鼠标灵敏度、触摸标定值和手势阈值。
- 首次运行自动选择唯一 A2854；多台时提示输入序号；暂时断开时每 2 秒重连，设备被移除时重新选择。
- 日志仅记录状态、事件类型和错误；探针报告使用脱敏设备标识。
- `Ctrl+C`、关闭控制台、断线和输入流结束统一释放模拟输入与设备资源。
- 发布为 `win-x64` 自包含单文件，不引入图形界面、MSIX 或 Windows App SDK。
- 更新 README、按键映射说明和第一阶段验证记录。

## 接口与默认行为

- Core 与平台层之间只传递 `RemoteEvent` 和 `InputAction`，便于在 GATT/HID 读取方式之间切换。
- 配置损坏时报告错误并回退到默认阈值和设备重选，不覆盖损坏文件。
- 鼠标移动采用标定后的归一化相对增量、分数累计和抖动过滤。
- 退出码：正常退出为 `0`，设备或验证条件不满足为 `1`，不可恢复错误为 `2`。

## 测试与验收

- 自动化测试覆盖报文解析、全部按键状态、长按计时、拖拽、旋转、双指轻碰取消条件、断线和退出释放、参数解析及配置读写。
- 执行 `dotnet build`、`dotnet test` 和自包含单文件发布验证。
- 使用实体 A2854 在 Windows 10 完成 AC-01～AC-20，包括按住期间断线、自动重连、Ctrl+C 清理及 10 次重连。
- 稳定性运行缩短为 **15 分钟**，期间观察崩溃、卡键、鼠标漂移、重连和明显资源持续增长，并保存结果。
- 验证记录明确注明：15 分钟测试仅属于快速稳定性测试，原需求中的 8 小时稳定性验收尚未执行，不能据此声明已经满足 8 小时稳定性指标。
- 交付记录同时注明：Windows 10 22H2 已测；Windows 11、8 小时稳定性及第二阶段语音功能均未在本轮验收。
