# Apple TV A2854 Windows 遥控器映射软件需求规格说明书

| 项目 | 内容 |
| --- | --- |
| 文档版本 | V1.1 |
| 文档日期 | 2026-07-23 |
| 目标设备 | Apple TV Siri Remote A2854（USB-C 版） |
| 目标平台 | Windows 10/11 64 位 |
| 软件形态 | 普通 Windows 桌面软件 |

## 1. 项目概述

### 1.1 项目目标

开发一款普通 Windows 桌面软件，将 Apple TV A2854 遥控器映射为 Windows 输入控制器，实现：

- 使用 Clickpad 触摸区域控制鼠标移动；
- 使用 Clickpad 中心按键执行鼠标左键单击和双击；
- 使用 Clickpad 圆环旋转控制鼠标滚轮；
- 使用双指轻碰 Clickpad 圆环区域执行鼠标右键单击；
- 使用圆环四方向按键发送 Windows 上、下、左、右方向键；
- 使用语音键进行按住说话，松开后识别语音并把文字输入当前输入框；
- 使用返回键、TV/控制中心键、静音键和音量键执行指定的 Windows 操作。

### 1.2 产品范围

本期仅支持 A2854 遥控器和本文件定义的映射功能。软件以普通用户权限在 Windows 桌面环境运行，不开发自定义蓝牙驱动、虚拟 HID 驱动或虚拟音频驱动。

以下内容不属于本期范围：

- 将遥控器注册为 Windows 系统麦克风；
- 控制 Windows 登录界面、锁屏界面或 UAC 安全桌面；
- 未在本文件中定义的电源键、播放/暂停键映射；
- 多套按键方案、按应用切换方案或宏命令。

## 2. 适用平台与运行条件

### 2.1 适用平台

- Windows 10 64 位；
- Windows 11 64 位；
- 支持 Bluetooth Low Energy（BLE）的电脑；
- Apple TV Siri Remote A2854。

### 2.2 前置条件

- A2854 已与 Windows 完成蓝牙配对并处于可连接状态；
- 软件能够读取遥控器的按键、触摸和语音相关数据；
- 使用语音输入前，用户已在目标应用中选中可输入文字的输入框；
- 目标应用与本软件处于相同或更低的 Windows 权限级别。

### 2.3 术语

- **Clickpad**：A2854 顶部圆形触摸点击区域，由中心区域和外侧圆环组成。
- **中心区域**：Clickpad 中间可按压、可触摸的区域。
- **圆环区域**：Clickpad 外侧支持四方向点击和旋转手势的环形区域。
- **一次触摸会话**：从手指首次接触 Clickpad 到所有手指离开 Clickpad 的完整过程。
- **当前输入框**：语音识别完成并准备输入文字时，Windows 当前拥有输入焦点的文本输入控件。

## 3. 总体功能要求

### 3.1 启动与运行

1. 软件启动后应开始监听已配对 A2854 的输入事件。
2. 遥控器可用时，软件应启用全部映射功能。
3. 遥控器断开时，软件不得继续产生鼠标、键盘或音量事件。
4. 遥控器重新连接后，软件应恢复事件监听，无需用户重启软件。
5. 软件关闭后应停止全部映射和语音处理。

### 3.2 事件处理原则

1. 每个物理按键的按下和松开状态应被正确区分。
2. 同一物理事件不得被重复映射为多个意外操作。
3. 遥控器断开、软件退出或输入流异常结束时，应清除全部未释放的按键和触摸状态。
4. 触摸移动使用相对位移，不将 Clickpad 坐标直接映射为屏幕绝对坐标。

### 4.1 Clickpad 中心按键

#### 4.1.1 单击

- 每次完整的中心按键按下和松开应生成一次鼠标左键单击。
- 软件应立即发送左键按下和左键松开事件，不额外等待双击判断窗口。

#### 4.1.2 双击

- 用户在 Windows 当前双击时间阈值内连续点击中心按键两次时，应被 Windows 识别为鼠标左键双击。
- 双击速度遵循 Windows 系统的鼠标双击速度设置，软件不单独维护另一套双击时间。

### 4.2 圆环四方向按键

- 按下圆环上、下、左、右方向时，分别发送对应 Windows 方向键按下事件。
- 松开物理方向键时，发送对应 Windows 方向键松开事件。
- 长按时应支持 Windows 正常的连续方向输入。

### 4.3 其他实体按键

- 返回键固定映射为 `Delete`，不是 `Backspace`。
- TV/控制中心键每次单击执行一次 `Alt+Tab`，切换到下一个软件。
- 静音键每次单击切换一次系统静音状态。
- 音量加、减键支持单击调节和长按连续调节。

## 5. 触摸与旋转判定逻辑

### 5.1 判定目标

中心区域和圆环区域共同构成一个连续触摸面。软件必须根据一次触摸会话的首次接触位置和触点数量判定本次手势模式，避免鼠标移动、滚轮和鼠标右键同时触发。

### 5.2 模式判定

一次触摸会话只允许完成以下一种映射：

| 首次接触位置与触点 | 模式或候选手势 | 后续行为 |
| --- | --- | --- |
| 一根手指从中心区域开始 | 鼠标移动模式 | 即使手指随后移动到圆环区域，仍继续控制鼠标移动 |
| 一根手指从圆环区域开始 | 圆环手势等待状态 | 单指产生有效旋转后进入旋转滚轮模式；第二根手指及时落入圆环后进入双指轻碰候选状态 |
| 两根手指均在圆环区域 | 双指轻碰候选状态 | 满足双指轻碰条件时执行鼠标右键单击 |
| 任一手指从中心区域开始 | 不得识别为双指轻碰圆环 | 按鼠标移动模式处理，或在无法保持单指轨迹时取消本次手势 |

### 5.3 状态流程

```text
无触摸
  ↓ Touch Down
读取首次接触位置
  ├─ 中心区域 → 锁定鼠标移动模式
  └─ 圆环区域 → 进入圆环手势等待状态
                    ├─ 单指产生有效旋转 → 锁定旋转滚轮模式
                    └─ 第二根手指落入圆环 → 双指轻碰候选状态
  ↓
持续处理当前模式
  ↓ Touch Up
判断候选手势或结束当前模式，然后清除状态
```

### 5.4 鼠标移动模式

1. 使用相邻触摸采样点的 X/Y 坐标差计算鼠标相对位移。
2. 手指在中心和圆环之间移动时不得切换为滚轮模式。
3. 手指离开后停止鼠标移动并清除上一个坐标。
4. 轻微触摸抖动不得造成明显的鼠标漂移。
5. 鼠标移动方向应与手指移动方向一致。

### 5.5 旋转滚轮模式

1. 仅使用单指相邻触摸采样点相对 Clickpad 中心的角度差判断旋转方向和距离。
2. 顺时针旋转映射为鼠标滚轮向下；逆时针旋转映射为鼠标滚轮向上。
3. 滚轮步数应随累计旋转角度增加，不因单个微小抖动频繁触发。
4. 手指在本次会话中滑入中心区域时，仍保持旋转滚轮模式。
5. 手指离开后结束滚轮模式，下一次触摸重新判断起点。

### 5.6 双指轻碰圆环模式

#### 5.6.1 有效手势条件

一次有效的双指轻碰圆环必须同时满足以下条件：

1. 两个触点首次出现的位置都在预先标定的圆环区域内；
2. 任一触点都不得从中心按键区域开始；
3. 第二根手指应在第一根手指接触后的 100 ms 内出现；
4. 从第一根手指接触到两根手指全部离开的总时间不得超过 250 ms；
5. 每个触点的移动距离不得超过 Clickpad 有效宽度的 5%；
6. 两根手指的离开时间差不得超过 100 ms；
7. 整个过程中不得出现 Clickpad 中心按键 `Select` 的物理按下事件；
8. 两个触点在离开前始终位于圆环区域内，不得进入中心按键区域。

#### 5.6.2 输出规则

- 满足全部条件时，软件发送一次鼠标右键按下和一次鼠标右键松开；
- 每次触摸会话最多产生一次鼠标右键单击；
- 双指轻碰候选状态期间不得产生鼠标移动或滚轮事件；
- 任一条件不满足时，取消本次双指轻碰，不发送鼠标右键事件；
- 候选手势取消后，应等待两根手指全部离开，再开始识别下一次触摸会话。

## 6. 语音输入流程

### 6.1 正常流程

```text
用户选中目标输入框
  ↓
按下语音键
  ↓
创建语音会话并开始接收遥控器音频
  ↓
用户持续按住语音键并讲话
  ↓
松开语音键
  ↓
停止接收音频并结束语音会话
  ↓
调用软件语音识别模块
  ↓
获得识别文字
  ↓
将文字输入当前拥有焦点的输入框
```

### 6.2 语音键按下

检测到语音键由松开变为按下时，软件应：

1. 创建新的语音输入会话；
2. 清空上一会话残留的音频和识别结果；
3. 初始化音频接收和解码；
4. 开始接收并缓存遥控器麦克风音频；
5. 将内部状态切换为“正在聆听”。

按键保持按下期间重复收到的按下事件不得创建多个语音会话。

### 6.3 语音键松开

检测到语音键由按下变为松开时，软件应：

1. 停止接收本次语音；
2. 完成本次音频数据处理；
3. 调用语音识别模块；
4. 取得最终识别文字；
5. 将非空结果作为键盘文本输入到当前输入框；
6. 结束并释放本次语音会话。

### 6.4 文字输入规则

- 仅输入语音识别返回的最终文字；
- 识别结果为空时不输入任何内容；
- 不自动发送 Enter；
- 不自动清除输入框中已有内容；
- 文字插入当前光标位置；
- 不要求向管理员权限应用、UAC、安全桌面或锁屏界面注入文字。

## 7. 异常与边界情况

| 场景 | 预期处理 |
| --- | --- |
| 软件启动时遥控器未连接 | 不产生映射事件；等待遥控器重新连接 |
| 使用过程中遥控器断开 | 立即停止当前触摸、按键和语音会话，并释放所有状态 |
| 遥控器重新连接 | 自动恢复输入监听 |
| 语音键快速点击但没有有效音频 | 结束会话，不调用文字输入 |
| 语音键按下时已有语音会话 | 忽略重复按下事件，不创建第二个会话 |
| 未收到语音键松开事件 | 遥控器断开或连续录音达到 60 秒时自动结束并取消本次会话 |
| 语音数据中断或无法解码 | 取消本次识别，不输入不完整文字，并提示语音输入失败 |
| 语音识别失败或返回空结果 | 不输入文字，并提示未识别到内容 |
| 识别期间输入框失去焦点 | 仅在识别完成时的当前焦点控件可输入时写入；否则不注入文字并提示用户重新选择输入框 |
| 触摸起点位于中心与圆环边界附近 | 按预设区域边界只判定一次，并锁定到手指离开 |
| 触摸过程中从中心跨到圆环 | 保持鼠标移动模式 |
| 单指已锁定旋转滚轮模式后从圆环跨到中心 | 保持旋转滚轮模式 |
| 双指轻碰时任一触点位于或进入中心区域 | 取消双指轻碰，不发送鼠标右键事件 |
| 双指轻碰期间任一触点移动超过阈值 | 取消双指轻碰，本次触摸会话不产生鼠标右键、滚轮或移动事件 |
| 双指轻碰期间物理按下 Clickpad | 按物理按键规则处理，不发送鼠标右键事件 |
| 第二根手指出现过晚或两根手指释放时间差过大 | 不识别为双指轻碰，不发送鼠标右键事件 |
| 触摸数据短暂丢失 | 不使用过期坐标计算大幅位移；恢复后以新采样点为起点 |
| 按键按下期间遥控器断开 | 补发相应松开状态，避免方向键或音量键持续按住 |
| 软件退出 | 停止监听并释放全部按键、触摸、音频及连接资源 |

## 8. 非功能需求

### 8.1 性能

- 普通按键从接收到生成 Windows 输入事件的延迟目标不超过 50 ms；
- 触摸移动应连续，无明显跳动或肉眼可感知的额外停顿；
- 有效双指轻碰结束后应在 100 ms 内生成鼠标右键单击；
- 语音键按下后应在 200 ms 内进入“正在聆听”状态；
- 在验收测试环境中，5 秒普通话音频在松开语音键后 5 秒内完成识别并开始输入文字。

### 8.2 稳定性

- 软件连续运行 8 小时不应崩溃或出现持续的卡键、漂移；
- 遥控器连续断开和重连 10 次后，映射功能仍应正常；
- 单次输入或识别失败不得导致软件退出。

### 8.3 易用性

- 软件启动后，在遥控器已配对的情况下无需重复设置即可使用；
- 连接、聆听、识别中和失败状态应有简洁、可区分的界面或托盘提示；
- 错误提示应说明失败原因和用户可执行的下一步操作。

### 8.4 兼容性

- 支持 Windows 10/11 64 位桌面环境；
- 支持常见文本输入场景，包括记事本、浏览器输入框和普通桌面应用输入框；
- 不承诺支持管理员权限窗口、安全桌面、远程桌面特殊输入通道或使用专有输入机制的应用。

### 8.5 隐私与数据

- 默认不长期保存原始语音音频；
- 默认不在日志中记录完整识别文字或原始音频；
- 如果语音识别需要把音频发送到网络服务，应在软件中明确告知用户。

| AC-13 | 单击 TV/控制中心键 | Windows 切换到下一个软件 |
| AC-14 | 单击静音键 | Windows 系统静音状态切换一次 |
| AC-15 | 单击及长按音量加、减键 | 系统音量按对应方向单步或连续变化 |
| AC-16 | 在记事本输入区按住语音键说一句普通话后松开 | 开始和结束语音会话，识别文字输入当前光标位置 |
| AC-17 | 松开语音键后得到空识别结果 | 输入框内容不变，不自动发送 Enter |
| AC-18 | 语音过程中断开遥控器 | 语音会话取消，软件不崩溃、不输入不完整文字 |
| AC-19 | 触摸或按键按住期间断开遥控器 | 鼠标停止，所有按键状态被释放，无卡键 |
| AC-20 | 遥控器断开后重新连接 | 映射功能自动恢复 |

### 9.2 验收通过条件

- 所有 P0 验收项必须通过；
- 不得存在导致软件崩溃、持续卡键、持续鼠标漂移或错误输入文字的阻断问题；
- P1 项目允许存在不影响核心操作的显示或调参问题，但须记录并安排修复。

## 10. 开发优先级

### 10.1 P0：可行性验证

在完整开发前先完成最小验证程序，确认目标 Windows 环境能够读取：

1. 全部实体按键的按下和松开事件；
2. Clickpad 单指和双指触摸坐标、触点数量及首次接触位置；
3. 圆环旋转所需的连续触摸数据；
4. 语音键按下和松开事件；
5. 遥控器语音数据，并能解码为可识别音频。

如果第 5 项无法通过普通 Windows API 获得，本期“纯普通桌面软件读取遥控器语音”的技术路线需重新评估；不得以电脑自带麦克风替代而宣称功能完成。

### 10.2 P0：核心功能

1. 遥控器连接、断开与自动恢复；
2. 中心按键左键单击和双击；
3. Clickpad 鼠标移动；
4. 圆环旋转滚轮；
5. 双指轻碰圆环映射鼠标右键单击；
6. 圆环四方向键；
7. 返回、TV/控制中心、静音和音量按键映射；
8. 语音键按下开始、松开结束；
9. 遥控器语音采集、识别及文字输入；
10. 断线、卡键和语音失败处理。

### 10.3 P1：体验完善

1. 鼠标移动速度、抖动过滤和滚轮步进调优；
2. 连接、聆听、识别中和失败状态提示；
