# Apple TV Siri Remote A2854 按键映射关系

## 1. 基本定义

- **Clickpad 触摸区域**：由中心区域和外侧圆环区域组成的完整连续触摸面。
- **短按**：按键按下后不足 600 ms 即松开。
- **长按**：按键持续按住达到 600 ms。
- **语音会话**：从语音键按下开始，到语音键松开或设备断开时结束的音频转发过程。

## 3. Clickpad 触摸与点击

### 3.1 鼠标移动

1. Clickpad 中心区域和圆环区域作为同一个触摸面处理。
2. 单指可以从 Clickpad 任意位置开始滑动，并可跨越中心与圆环边界。
3. 软件根据相邻触摸采样点的 X、Y 坐标差计算鼠标相对位移。
4. 单指在 Clickpad 上沿任意轨迹移动时，均按照二维相对位移控制鼠标指针。
5. 手指离开 Clickpad 后，本次触摸会话结束，并清除上一个触摸坐标。
6. Clickpad 仅使用单指触摸数据控制鼠标移动；多指触摸不产生映射输出。

### 3.2 中心按键

1. 中心按键短按一次时，发送一次鼠标左键按下和松开事件。
2. 在 Windows 当前鼠标双击时间内连续短按两次时，由 Windows 识别为鼠标左键双击。
3. 中心按键持续按住达到 600 ms 时，发送一次 `Enter` 键按下和松开事件。
4. 每次长按最多发送一次 `Enter`，继续按住不重复发送。
5. 长按已经触发后，松开中心按键时不产生鼠标左键单击。

### 3.3 圆环方向键

1. 按下圆环上、下、左、右位置时，分别发送对应方向键的按下事件。
2. 松开对应位置时，发送对应方向键的松开事件。
3. 持续按住时，按照 Windows 当前键盘重复设置产生连续方向输入。

## 4. 音量键

1. 音量加键或音量减键在按住时间不足 600 ms 时松开，调节一次 Windows 系统音量。
2. 音量加键按住达到 600 ms 后，持续发送鼠标滚轮向上事件。
3. 音量减键按住达到 600 ms 后，持续发送鼠标滚轮向下事件。
4. 松开音量键时，立即停止发送滚轮事件。
5. 一次按键过程只执行短按音量操作或长按滚轮操作中的一种。

## 5. 鼠标拖拽

1. TV/控制中心键在按住时间不足 600 ms 时松开，执行一次 `Alt+Tab`。
2. TV/控制中心键按住达到 600 ms 时，发送鼠标左键按下事件并进入拖拽状态。
3. 拖拽期间，在 Clickpad 触摸区域内单指滑动可移动鼠标指针。
4. 松开 TV/控制中心键时，发送鼠标左键松开事件并结束拖拽状态。
5. 每次长按只生成一组鼠标左键按下和松开事件。

## 6. 播放/暂停

1. 播放/暂停键短按时，发送一次 Windows 媒体播放/暂停键的按下和松开事件。
2. 播放/暂停键长按不产生额外操作。

## 7. 语音输入

### 7.1 工作方式

1. 按下语音键时，软件创建语音会话并开始接收遥控器的语音音频。
2. 软件实时解码音频，并转换为虚拟声卡支持的 PCM 格式。
3. 解码后的音频持续写入用户选择的虚拟声卡播放端点。
4. Windows 应用通过虚拟声卡对应的录音端点接收音频，并将其作为麦克风输入。
5. 松开语音键时，软件停止接收和转发音频，并释放语音会话资源。

### 7.2 VB-CABLE 配置示例

| 使用方 | 选择的端点 |
| --- | --- |
| 本软件 | `CABLE Input` 播放端点 |
| 需要使用语音的 Windows 应用 | `CABLE Output` 录音端点 |

虚拟声卡由用户预先安装和配置。本软件只负责本机音频接收、解码和转发。

## 8. Windows 唤醒

1. 遥控器保持与电脑配对，蓝牙适配器和设备驱动允许该设备唤醒电脑。
2. Windows 进入睡眠或休眠状态后，按下遥控器任意实体按键可触发系统唤醒。
3. 用于唤醒的第一次按键只执行唤醒操作，不执行该按键在正常运行状态下的映射。
4. Windows 恢复运行后，软件重新连接遥控器并恢复全部映射功能。
5. 唤醒能力取决于电脑硬件、蓝牙适配器、驱动程序和 Windows 电源管理设置。

## 9. 状态释放

- 遥控器断开或软件退出时，立即停止触摸、按键、拖拽、滚轮和语音会话。
- 所有已经发送的键盘按下和鼠标按下状态都必须发送对应的松开事件。
- 遥控器重新连接后，从空闲状态恢复映射，不沿用断开前的触摸坐标或按键状态。
